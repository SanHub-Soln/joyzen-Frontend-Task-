# joyzen-Frontend-Task-
# Project README

Hi

This task was really interesting for me. I chose React along with Tailwind for styling and converted the Figma design into a React application. I did not want it to feel like a static page. Static designs look clean but they do not always engage the user. I wanted to bring life into the UI with interactions, so I decided to use Framer Motion along with React.

## My Approach

My main thought was simple
a user should not just see the page
they should feel it and interact with it

I believe good UI is not only about structure but also about movement and experience.

## What I Added

I added a few interactive elements to make the experience more engaging

Preloader
I created an eye ball movement preloader where a ball is balanced dynamically. This animation runs until the page finishes loading. The idea was to give a playful and engaging first impression.

Scroll Reveal Text
Instead of showing all content at once I added smooth scroll based text reveal. This helps users focus and read content step by step while also making the experience feel smooth.

Grid Reveal Animation
For the grid sections I used a reveal effect. My thought was to avoid a plain layout. I wanted it to feel like discovering something piece by piece similar to finding nuts inside chocolate rather than seeing everything at once.

Mobile Experience
For mobile view I used a carousel like movement for better usability. This makes scrolling feel more controlled and natural on smaller screens.

Call to Action Interaction
At the bottom I added a hover animation inspired by the same ball balancing concept used in the preloader. When hovered it animates and then returns to its original position which keeps the interaction consistent across the app.

## Design Fidelity

Most of the design follows the Figma file closely. The main difference is the added interactions and animations to enhance the experience. I also used images wherever required as per the design.

## Responsiveness

I have tested the layout for responsiveness across different screen sizes and tried to ensure a smooth experience on both desktop and mobile.

## Final Thought

I do not prefer static web pages. I always try to add interaction so that the user feels curious and continues exploring. My goal was to guide the user attention through movement and make them naturally scroll and explore the full content.

This project reflects that idea.
